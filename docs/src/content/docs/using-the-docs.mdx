## How each track is grouped

Every track separates learning-oriented pages from reference:

| Group | What it is | How to read it |
| --- | --- | --- |
| **Get started** | A tutorial path | In order, top to bottom |
| **Guides / How-to** | Task-focused recipes | Jump to the one you need |
| **Reference** | Exhaustive look-up | Search / scan, don't read cover to cover |
| **Deep dives** | Implementation internals | When you want the "why" and the sharp edges |

The Protocol Reference adds a few more groups (_The core model_, _The planes_,
_Moving &amp; reconciling_, _Subsystems_) because it's a sequence — it follows one
operation from birth to convergence.

## Page conventions

- **Callouts.** `:::note` is context, `:::tip` is a shortcut, `:::caution` is a
  footgun. In the Protocol spec, a **`Normative`** caution marks a MUST/SHOULD
  requirement an independent implementation has to satisfy.
- **Diagrams** are animated — hit **▶ Replay** to step through a flow again.
- **Code blocks** have a copy button; spec sections give exact byte layouts.
- **Pre-1.0.** Everything here is accurate to current code but may change before
  1.0; the byte-level details are versioned (see the spec's versioning note).
