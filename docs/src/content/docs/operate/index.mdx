A node is one `merod` process: it holds keys, runs applications, stores state,
and gossips with peers. You drive it with the `meroctl` CLI and the admin API.

## Start here

<Steps>

1. **[Install &amp; first run](/operate/install/)** — build/Docker, then `init` → `run` → verify.
2. Learn the **[`merod`](/operate/merod/)** and **[`meroctl`](/operate/meroctl/)** commands.
3. When you go to production, work through the **Guides** (deployment, networking, security).
4. Keep **troubleshooting** and the **runbooks** handy for operations.

</Steps>
